# Product Overview

AI-DLC (AI-Driven Development Life Cycle) is an intelligent software development methodology that adapts to project needs while maintaining quality standards and user control.

## Core Purpose

AI-DLC provides structured workflows for AI-assisted software development through three adaptive phases:
- **Inception Phase**: Planning and architecture (WHAT and WHY to build)
- **Construction Phase**: Design and implementation (HOW to build it)
- **Operations Phase**: Deployment and monitoring (placeholder for future expansion)

## Key Principles

- **Adaptive Intelligence**: Only executes stages that add value to the specific request
- **Context-Aware**: Analyzes existing codebase and complexity requirements
- **Risk-Based**: Complex changes get comprehensive treatment, simple changes stay efficient
- **Human in the Loop**: Critical decisions require explicit user confirmation
- **Methodology First**: Platform-agnostic approach that works with any IDE, agent, or model

## Platform Support

AI-DLC rules can be deployed to:
- Amazon Q Developer IDE Plugin/Extension (via `.amazonq/rules/`)
- Kiro CLI and IDE (via `.kiro/steering/`)
- Other AI coding assistants supporting similar rule/steering mechanisms

## Tenets

- **No duplication**: Single source of truth lives in one place
- **Methodology first**: Users shouldn't need to install anything to get started
- **Reproducible**: Rules should produce similar outcomes across different models
- **Agnostic**: Works with any IDE, agent, or model
- **Human in the loop**: Critical decisions require explicit user confirmation
